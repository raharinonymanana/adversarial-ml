# Lab 3: Offline Model Extraction

**Course:** NVIDIA DLI — Exploring Adversarial Machine Learning
**Category:** Extraction (model stealing) attacks
**Victim model:** small neural net (`MLPClassifier`, hard labels only)
**Clone model:** Random Forest (`RandomForestClassifier`) — different internals on purpose
**Dataset:** scikit-learn built-in handwritten digits (`load_digits`, 1,797 × 8×8 images)
**Hardware:** CPU-only, no GPU required

---

## The Idea: A Stolen Model Is a Portable Asset

Lab 2 showed you can clone a black-box model using only its predictions. This lab
drives home **why that is so dangerous**: once the attacker has a clone, it is a
**standalone file**. It no longer needs the victim, the API, or even the
internet. The punch line is `offline_inference.py`, which classifies digits with
the **network physically disabled** — if any line tried to phone home, the script
would crash. It doesn't crash, so the clone is provably self-contained.

## Three Phases

```
  PHASE 1 — BUILD THE VICTIM            PHASE 2 — STEAL IT
  (train_victim.py)                     (extract.py)
  ┌──────────────────────────┐         ┌──────────────────────────┐
  │ digits ─► MLP victim      │         │ unlabeled pool            │
  │ saved as victim_mlp.pkl   │         │      │                    │
  │                           │         │      ▼  1,000 queries     │
  │ exposes ONE black box:    │ ──────► │ victim_predict(X) ─► labels│
  │   victim_predict(X)       │  query  │      │                    │
  │                           │  only   │      ▼                    │
  │ (weights/code never seen) │         │ train Random Forest clone │
  └──────────────────────────┘         │ saved as clone_rf.pkl     │
                                        └──────────────────────────┘
                                                   │
                                                   ▼
  PHASE 3 — GO OFFLINE  (offline_inference.py)
  ┌────────────────────────────────────────────────────────────┐
  │  ✂  NETWORK CABLE CUT  (socket calls patched to raise)       │
  │                                                              │
  │  load clone_rf.pkl  ─►  clone.predict(digits)  ─►  labels    │
  │                                                              │
  │  prints "[PASS] predicted with network disabled"            │
  │  (any attempt to use the network would crash the script)     │
  └────────────────────────────────────────────────────────────┘
```

The only channel between Phase 1 and Phase 2 is `victim_predict(...)` — black-box
queries. Phase 3 never touches the victim at all; it only reads `clone_rf.pkl`.

## Why a Disjoint Three-Way Split

`train_victim.py` carves the 1,797 digits into **three slices that never overlap**:

| Slice | Size | Who uses it |
|---|---|---|
| victim-train | 497 | the victim learns only from this |
| attacker-pool | 1,000 | the attacker's **unlabeled** images (labels discarded) |
| held-out test | 300 | nobody trains on it; used to score everyone |

Keeping them disjoint is what makes the numbers **honest**: the clone is judged on
images it never saw during the attack, so its score can't be inflated by leakage.

---

## Success Criteria

A single extraction reports one number, and that number wobbles depending on which
images were queried and how the forest grew. So `extract.py` repeats the **whole
attack across five seeds** (`SEEDS = [42, 43, 44, 45, 46]`), varying *only the
attack*, and `evaluate.py` reports **mean ± std**. It judges the **mean** against
each target — never a bare PASS/FAIL, which would just invite tuning the attack
until one lucky run crosses a line.

| Metric | Meaning | Target |
|---|---|---|
| **Clone accuracy** | clone correctness vs the TRUE digit labels | **~95%** |
| **Clone agreement** | % of test images where the clone matches the **victim** | **~90%** |

Verdict per metric: `PASS` if the mean meets the target, `≈ within ±1σ` if it
falls short by less than one standard deviation, or `BELOW` if it misses by more.

Typical run *(deterministic with the seeds above)*:

```
====================================================================
  OFFLINE MODEL EXTRACTION RESULTS - Handwritten Digits
====================================================================

  Attack repeated over 5 seeds : [42, 43, 44, 45, 46]
  Budget per run            : 1,000 black-box victim queries
  Victim (MLP) test accuracy: 96.00%  (fixed across all seeds)

Metric                          mean ± std    Target         Verdict
--------------------------------------------------------------------
Clone accuracy (vs truth)   94.47% ± 0.61%      ~95%    ≈ within ±1σ
Clone agreement (vs victim)  92.73% ± 0.55%      ~90%            PASS
--------------------------------------------------------------------

  Measurement floor: n=300 test set => ~±2% binomial CI; per-seed
  std above captures attack variance, a separate source.
```

And the offline proof:

```
[OK] network guard live - socket creation raised
...
  [PASS] predicted with network disabled
  The stolen clone is a self-contained asset: no API, no
  victim, no internet - just a pickle file and a CPU.
```

### Reading the numbers

- **One victim, many attacks.** We hold the victim **and** the test set fixed and
  vary only the attack across the five seeds, because a real attacker faces a
  single deployed model. The question is how much *their* luck — which images they
  happen to query, how their forest happens to grow — moves the result, not how a
  differently-trained victim would behave.
- **Two different noise sources.** The per-seed **std** (~0.6%) measures *attack
  variance*. Separately, scoring on only **n = 300** test images carries a
  *measurement* uncertainty of about **±2%** (a binomial confidence interval).
  These are distinct things: one is "how much the attack wobbles run-to-run," the
  other is "how precisely 300 examples can pin down any percentage at all."
- **Why accuracy (~94%) sits above agreement (~93%).** The attacker queried *real*
  unlabeled digits, so the clone partly learned the **true task itself**, not only
  the victim's decision boundary — accuracy measures the former, agreement the
  latter. Accuracy exceeding agreement is the accuracy-vs-fidelity split, live.

> **Caveat:** five seeds is a *small* sample for estimating a standard deviation —
> treat the ± figures as illustrative of the spread, not a definitive error bar.
> The honest takeaway: agreement clears its target comfortably, while accuracy
> hovers right around 95%, well within the normal run-to-run spread.

## How `offline_inference.py` Proves "Offline"

Before importing anything heavy or loading any file, the script monkeypatches
Python's `socket` module so that **opening any network connection raises
`RuntimeError`**:

- `socket.socket` is replaced with a subclass whose constructor refuses to run.
  (It must stay a *subclass-able class* because the standard library defines
  `class SSLSocket(socket)` at import time — a plain function there would crash
  importing for the wrong reason.)
- `socket.create_connection` is replaced with a function that raises.

It then **proves the guard is live, not a silent no-op**: it tries to create a
socket and confirms that raises. If a socket somehow *succeeded*, the patch failed
and the offline claim would be meaningless, so the script aborts right there. Only
once the guard is verified does it load `clone_rf.pkl` with plain local file I/O
and classify digits. The script **never imports `train_victim`** and never loads
the victim — the clone stands entirely on its own. If it finishes and prints
`[PASS]`, that is proof the prediction needed zero network access.

## Why Offline Theft Is the Real Threat

A live API can be watched and throttled. A **stolen offline clone cannot**:

- **No more rate limits.** Defenses like query quotas, anomaly detection, and
  per-user throttling all live at the API. Once the model is local, every one of
  them is bypassed — the attacker can run it a billion times for free.
- **Unlimited offline attack crafting.** With a local copy, an attacker can
  hammer the clone to design **evasion attacks** (Lab 1) at zero risk of
  detection, then fire the finished attacks at the real system.
- **Persistence.** Revoking the attacker's API key does nothing — they already
  walked away with a working model in a single `.pkl` file.
- **Redistribution.** A standalone clone can be copied, sold, or embedded in
  another product. The victim's intellectual property has effectively leaked.

The lesson: protecting the *endpoint* is not enough. If a faithful clone can be
extracted at all, defenders must assume it will eventually run somewhere they
can't see.

### Defenses (for further study)

| Defense | Idea |
|---|---|
| **Rate limiting / quotas** | Make collecting a big stolen dataset slow and costly — but useless once a clone already exists. |
| **Query monitoring** | Flag clients whose queries look like systematic boundary-probing instead of normal traffic. |
| **Output perturbation** | Return coarser or noisier labels/scores so each query leaks less. |
| **Watermarking** | Plant secret input→output behaviours so a stolen clone can be proven to be a copy. |
| **Prediction privacy** | Return only top-1 hard labels (as here) rather than full probability vectors. |
